<anti_patterns>
**Do NOT hard-fail on the FIRST run when bootstrap is missing.** Per REQ-10-07: graceful first-run, hard-fail subsequent. The marker file `.planning/.flutter-ui-bootstrap-done` tracks "have we already run setup once?" Missing pubspec/integration_test + no marker → warn-only setup task. Missing pubspec/integration_test + marker exists → hard fail.

**Do NOT auto-commit the pubspec changes silently.** Per RESEARCH.md Open Question 3: tag the setup task with `caution="pause-before-destructive"` so the user reviews and confirms.

**Do NOT skip the test_driver/integration_test.dart scaffold step.** TRD 10-04b's executor uses `flutter drive --driver=test_driver/integration_test.dart` for WEB verification. Without this driver file, `flutter drive` cannot invoke `tests.integration` on web. The bootstrap setup task MUST scaffold this file.

**Do NOT add a maestro_web opt-in to the setup task.** Maestro is mobile-only by design. Web verification flows through `flutter drive` (see TRD 10-02's "Web verification mechanism" section). The bootstrap creates `integration_test/`, `.maestro/`, and `test_driver/integration_test.dart` — no maestro_web flag.

**Do NOT generate test fixtures with an LLM.** Bootstrap test fixtures (a stub pubspec, an empty integration_test dir) are hand-built via temp dirs in the test setup function.
</anti_patterns>

<gotchas>
- Bootstrap detector reads `pubspec.yaml` content for the `integration_test:` dev_dep entry — this is a YAML grep, not full YAML parse. Pattern: `/^\s+integration_test\s*:\s*\n\s+sdk\s*:\s*flutter\s*$/m`.

## GREEN phase (commit 2: `feat(10-04a): implement flutter-ui-bootstrap detector (GREEN)`)

Create `flutter-ui-bootstrap.cjs`:

```javascript
'use strict';

const fs = require('fs');
const path = require('path');
const { output } = require('./helpers.cjs');

const INTEGRATION_TEST_DEP_RE = /^\s+integration_test\s*:\s*\n\s+sdk\s*:\s*flutter\s*$/m;

/**
 * Pure function: given a project dir, check Flutter UI bootstrap state.
 * Returns { ready, missing, action, setup_task? }.
 *
 * action:
 *   'skip' — all good
 *   'warn' — missing but no marker (first run)
 *   'fail' — missing + marker exists (subsequent run; hard fail)
 *
 * The setup_task field (present only when action:'warn') contains an XML task block
 * the executor (TRD 10-04b) inserts ahead of all other tasks. The block includes:
 * - pubspec edit to add integration_test dev_dep
 * - integration_test/ and .maestro/ directory creation
 * - test_driver/integration_test.dart scaffold (required for web `flutter drive` per TRD 10-04b)
 * - marker file creation (LAST — atomic semantics)
 */
function checkBootstrapState({ projectDir }) {
  if (!projectDir) {
    return { ready: false, missing: ['unknown'], action: 'fail', error: 'projectDir required' };
  }
  const pubspecPath = path.join(projectDir, 'pubspec.yaml');
  const integrationTestDir = path.join(projectDir, 'integration_test');
  const maestroDir = path.join(projectDir, '.maestro');
  const markerPath = path.join(projectDir, '.planning', '.flutter-ui-bootstrap-done');

  const missing = [];

  // Check pubspec dev_dep
  if (!fs.existsSync(pubspecPath)) {
    missing.push('integration_test_dep');
  } else {
    const pubspecContent = fs.readFileSync(pubspecPath, 'utf-8');
    if (!INTEGRATION_TEST_DEP_RE.test(pubspecContent)) {
      missing.push('integration_test_dep');
    }
  }

  if (!fs.existsSync(integrationTestDir)) missing.push('integration_test_dir');
  if (!fs.existsSync(maestroDir)) missing.push('maestro_dir');

  const ready = missing.length === 0;
  const markerExists = fs.existsSync(markerPath);

  let action;
  if (ready) {
    action = 'skip';
  } else if (markerExists) {
    action = 'fail';
  } else {
    action = 'warn';
  }

  const result = { ready, missing, action };

  if (action === 'warn') {
    result.setup_task = `<!-- Auto-emitted Flutter UI bootstrap setup task -->
<task type="auto" caution="pause-before-destructive">
  <name>Bootstrap Flutter UI testing infrastructure</name>
  <files>pubspec.yaml, integration_test/.gitkeep, .maestro/.gitkeep, test_driver/integration_test.dart, .planning/.flutter-ui-bootstrap-done</files>
  <action>
First-time setup for Flutter UI verification:

1. Add to pubspec.yaml dev_dependencies (preserve other dev_dependencies):
   \`\`\`yaml
   dev_dependencies:
     flutter_test:
       sdk: flutter
     integration_test:    # NEW
       sdk: flutter       # NEW
   \`\`\`

2. \`mkdir -p integration_test && touch integration_test/.gitkeep\`

3. \`mkdir -p .maestro && touch .maestro/.gitkeep\`

4. Scaffold \`test_driver/integration_test.dart\` for Flutter web E2E (REQUIRED — the executor in TRD 10-04b uses
   \`flutter drive --driver=test_driver/integration_test.dart\` to run \`tests.integration\` on web):
   \`\`\`dart
   import 'package:integration_test/integration_test_driver.dart';
   Future<void> main() => integrationDriver();
   \`\`\`

5. \`flutter pub get\` to install the new dev dep.

6. ONLY AFTER all of 1-5 succeed: touch \`.planning/.flutter-ui-bootstrap-done\` to mark bootstrap complete. This marker triggers HARD FAIL on future runs if any of the above goes missing.

The caution attribute pauses execution before this task lands so the user can review the proposed pubspec diff.
  </action>
  <verify>
test -f .planning/.flutter-ui-bootstrap-done && \\
  test -d integration_test && \\
  test -d .maestro && \\
  test -f test_driver/integration_test.dart && \\
  grep -q 'integration_test:' pubspec.yaml && echo OK
  </verify>
  <done>Flutter UI verification infrastructure present; future TRD executions skip bootstrap.</done>
</task>`;
  }

  return result;
}

function cmdVerifyFlutterUIBootstrap(cwd, projectDir, raw) {
  const target = projectDir ? (path.isAbsolute(projectDir) ? projectDir : path.join(cwd, projectDir)) : cwd;
  const result = checkBootstrapState({ projectDir: target });
  output(result, raw);
}

module.exports = { checkBootstrapState, cmdVerifyFlutterUIBootstrap };
```
