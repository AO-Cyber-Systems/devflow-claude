<objective>
Build the planner-side gate for Flutter UI TRDs:

1. Create `lib/flutter-ui-scope.cjs` — a pure detector modeled on `lib/novel-domain.cjs`. Inputs: drafted TRD `files_modified` arrays, pubspec.yaml content, project-root file presence. Outputs: `{ detected: bool, signals: [...], evidence: [...] }`.
2. Wire it into `df-tools detect flutter-ui-scope <objective>` following the existing `detect novel-domain` registration pattern.
3. Edit `agents/planner.md` `<step name="break_into_tasks">` to invoke the detector AFTER initial breakdown, and:
   - When detected, set `type: ui`, `stack: flutter`, `platform: [mobile, web]` on each TRD touching `lib/**/*.dart`.
   - Detect `state_management:` from import scanning.
   - For each `type: ui` artifact, semantically require non-empty `states:`, `tests.widget`, `tests.integration`, `tests.maestro`.
   - On any missing field, emit `## PLANNING INCONCLUSIVE` (per-artifact, per-field), halt, return to orchestrator.

Purpose: This is the FIRST enforcement gate. Without it, downstream TRDs 10-04a/b (executor) and 10-05 (verifier) have no `type: ui` TRDs to act on — they'd be dormant. This TRD ensures every Flutter UI objective auto-acquires the schema fields the rest of the pipeline expects.

Why TDD: The detector is pure logic with clear input/output pairs. Each signal (lib-dart-files / pubspec-flutter-dep / description-keywords) and each derived field (platform / state_management) gets its own RED-GREEN cycle.

function detectFlutterUIScope({ trdFiles, pubspecContent, projectFiles }) {
  const signals = {
    lib_dart_files: detectLibDartFiles(trdFiles),
    pubspec_flutter_dep: detectPubspecFlutter(pubspecContent),
    flutter_keywords: detectFlutterKeywords(trdFiles),  // greps TRD bodies for flutter/widget/Riverpod/Bloc
  };
  const detected = Object.values(signals).some(s => s.fired);
  // Derived fields (only meaningful when detected) — platform ALWAYS defaults to [mobile, web].
  const platform = detected ? derivePlatform(pubspecContent) : null;
  return { detected, signals, platform };
}

<anti_patterns>
**Do NOT force the detector to run in non-Flutter projects.** The detector must be cheap and silent on non-Flutter objectives. Failsafe path: if pubspec.yaml is missing AND no drafted TRD touches `lib/**/*.dart`, return `{ detected: false }` immediately. No noise on the >90% of objectives in devflow-claude that don't touch Flutter.

**Do NOT add Patrol, Provider (non-Riverpod), MobX, or other state-management libraries.** Per user constraint: Riverpod, flutter_bloc, setState ONLY.

**Do NOT make new fields REQUIRED in FRONTMATTER_SCHEMAS.trd (defer to TRD 10-01 — that TRD explicitly does NOT change the schema).** This planner-side gate is the enforcement: PLANNING INCONCLUSIVE on missing fields. Schema stays permissive.

**Do NOT gate `platform: [mobile, web]` on the pubspec `platforms.web` block.** Per user correction (2026-05-24), both mobile and web are required coverage. `derivePlatform` returns `['mobile', 'web']` whenever Flutter UI scope is detected, regardless of pubspec's `platforms:` declarations. A consumer project that hasn't enabled web in its pubspec will still be required to add it (or to narrow the TRD's `platform:` field explicitly with documented justification).

**Do NOT introduce a `maestro_web` field anywhere.** Maestro is mobile-only by design. Web verification flows through `flutter drive` invoking `tests.integration` (see TRD 10-02's "Web verification mechanism" section). The planner emits PLANNING INCONCLUSIVE on missing `tests.integration` because BOTH platforms need it.

**Do NOT generate test fixtures with an LLM.** Per `no_llm_test_data` and CLAUDE.md TDD Playbook habit 4: pubspec.yaml fixtures and TRD fixtures are hand-built strings or hand-built files under `__fixtures__/flutter-ui-scope/`.
</anti_patterns>

<gotchas>
- The detector runs against drafted TRDs ANNOTATED IN MEMORY by the planner — they aren't on disk yet at detection time. Planner must pass the drafted `files_modified` arrays as input. The df-tools command line variant (`df-tools detect flutter-ui-scope <objective>`) reads on-disk TRDs as a secondary harness for testing; the planner's in-memory path is the primary path.
- pubspec.yaml may be in different locations depending on workflow:
  - For devflow-claude itself: no pubspec exists → detector should silently return `{ detected: false }`.
  - For a consumer project executing devflow against itself: pubspec is at `cwd`.
  - For a planner running against eden-ui-flutter from devflow-claude: pubspec is in the OTHER repo. RESEARCH.md Open Question 4 addresses this — default behavior is to scan `cwd` for pubspec. Cross-repo consumer-project scanning is deferred.
- `state_management: other` is a legitimate exit. Don't force every project to be Riverpod/Bloc/setState. Per RESEARCH.md anti-pattern: "Hardcoding Riverpod-only or Bloc-only patterns. The catalog must cover all three (Riverpod, Bloc, setState) or the detector returns `state_management: other` and gracefully degrades to truths-only coverage."
- The planner's `<step name="break_into_tasks">` is short (10 lines currently). The new Flutter UI sub-procedure adds ~40 lines and changes the flow. Keep the new content in a clearly-delimited block (`## Flutter UI scope sub-procedure (REQ-10-03)`) so future maintainers can find it.
- `derivePlatform` accepts `pubspecContent` as input but ALWAYS returns `['mobile', 'web']` when Flutter UI scope is detected. The parameter is kept for future extensibility (e.g., if a TRD's frontmatter says `platform: [mobile]` explicitly the planner can pass that through), but the DEFAULT behavior is hardcoded both-platforms. The `pubspec-web-enabled.yaml` fixture exists only for regression coverage (Case D12) — proves the function still returns `['mobile', 'web']` even when the legacy `platforms.web` block is present.
</gotchas>

const LIB_DART_RE = /^lib\/.*\.dart$/;

function detectLibDartFiles(trdFiles) {
  if (!Array.isArray(trdFiles)) return { fired: false, matches: [] };
  const matches = trdFiles.filter(f => LIB_DART_RE.test(f));

  if (!pubspecContent || typeof pubspecContent !== 'string') {

function deriveStateManagement(fileContents) {

  const joined = fileContents.join('\n');
  if (/package:flutter_riverpod/.test(joined)) return 'riverpod';

  const noPubspec = !pubspecContent || pubspecContent === '';

function cmdDetectFlutterUIScope(cwd, objectiveArg, raw) {
  // Failsafe: no objective argument
  if (!objectiveArg) {
    output({ detected: false, error: 'objective argument required' }, raw);
    return;
  }

  // Find objective directory
  const objectivesRoot = path.join(cwd, '.planning', 'objectives');
  if (!fs.existsSync(objectivesRoot)) {
    output({ detected: false, error: `no .planning/objectives directory in ${cwd}` }, raw);
    return;
  }

  // Find directory matching the objective number (e.g., "99-*")
  const padded = String(objectiveArg).padStart(2, '0');
  const dirs = fs.readdirSync(objectivesRoot).filter(d => d.startsWith(padded + '-') || d === objectiveArg);
  if (dirs.length === 0) {
    output({ detected: false, error: `no objective directory matching ${objectiveArg}` }, raw);
    return;
  }
  const objDir = path.join(objectivesRoot, dirs[0]);

  // Gather TRD files_modified from all TRDs in the directory
  const trdFiles = [];
  const descriptions = [];
  for (const f of fs.readdirSync(objDir)) {
    if (f.endsWith('-TRD.md')) {
      const trdContent = fs.readFileSync(path.join(objDir, f), 'utf-8');
      const fmMatch = trdContent.match(/^---\n([\s\S]+?)\n---/);
      if (fmMatch) {
        // Crude files_modified extraction — match `files_modified:` block + subsequent indented `-` lines
        const fmBody = fmMatch[1];
        const filesBlock = fmBody.match(/files_modified:\s*\n((?:\s+-\s+.+\n?)+)/);
        if (filesBlock) {
          for (const line of filesBlock[1].split('\n')) {
            const m = line.match(/^\s+-\s+(.+)$/);
            if (m) trdFiles.push(m[1].trim());
          }
        }
        // Also handle inline array shape: files_modified: [a, b]
        const inlineBlock = fmBody.match(/files_modified:\s*\[([^\]]+)\]/);
        if (inlineBlock) {
          inlineBlock[1].split(',').forEach(p => trdFiles.push(p.trim()));
        }
      }
      descriptions.push(trdContent);
    }
  }

  // Read pubspec at cwd (devflow-claude itself has none — falls through to other signals)
  const pubspecPath = path.join(cwd, 'pubspec.yaml');
  const pubspecContent = fs.existsSync(pubspecPath) ? fs.readFileSync(pubspecPath, 'utf-8') : '';

  // Read source-file contents for state_management detection (any .dart file in trdFiles that exists)
  const fileContents = [];
  for (const f of trdFiles) {
    if (f.endsWith('.dart')) {
      const abs = path.isAbsolute(f) ? f : path.join(cwd, f);
      if (fs.existsSync(abs)) {
        fileContents.push(fs.readFileSync(abs, 'utf-8'));
      }
    }
  }

  const result = detectFlutterUIScope({ trdFiles, pubspecContent, descriptions, fileContents });
  result.evidence = { objDir, trdFiles_count: trdFiles.length };
  output(result, raw);
}
