<objective>
Auto-generate a 1-page UAT checklist for Flutter UI objectives. For each `type: ui` TRD in the objective:
- One checklist row per `(must_haves.artifacts[*].states[*], platform)` pair — e.g., a TRD with platform=[mobile, web] and 3 states per artifact produces 6 state-coverage rows per artifact.
- One checklist row per `must_haves.artifacts[*].tests.maestro` flow — mobile-only (Maestro is mobile-only by design).
- One web-integration row per artifact when web is in platform — instructs user to run `flutter drive` and confirm pass.

The user walks through this checklist in ~5 minutes before marking the objective complete. The verifier (TRD 10-05 Step 9) invokes this at end of verification.

Purpose: Closes the human-verification loop. Static analysis (TRDs 10-04b, 10-05) catches a lot, but real-world rendering bugs (wrong colors, missing animations, broken layout) require human eyes. A pre-derived 5-minute checklist is far less friction than asking the user to invent their own test plan.

Output: New `lib/uat-generator.cjs` (~100 lines) + paired test (~140 lines) + df-tools subcommand + templates/UAT.md documentation update + verifier.md Step 9 hook.

Why TDD: The generator is pure logic — input is a list of TRDs + maestro flows, output is a deterministic markdown string. Every input shape variation gets its own RED-GREEN cycle.
</objective>

<anti_patterns>
**Do NOT clobber an existing user-edited UAT.md.** If `.planning/objectives/<obj>/<obj>-UAT.md` already exists with `status: testing` or `status: complete` or any non-pending results, the generator must REFUSE to overwrite (exit cleanly with a notice). Only overwrite when no UAT.md exists OR all results are still `[pending]` AND no manual edits detected.

**Do NOT include test rows for non-Flutter-UI TRDs in the objective.** Only `type: ui` + `stack: flutter` TRDs contribute rows. A mixed objective (some Node TRDs + some Flutter TRDs) generates UAT entries only from the Flutter ones.

**Do NOT gate web rows on any maestro_web opt-in flag.** Maestro is mobile-only BY DESIGN; there is no `maestro_web: true` opt-in in the schema. Web verification is achieved via `flutter drive` invoking `tests.integration`. If a TRD has `web` in its `platform:` field, the generator emits web rows by default (state-coverage rows × platform AND a flutter-drive instruction row per artifact).

**Do NOT generate test data with an LLM.** The TRD fixtures driving the generator tests are hand-built temp files.
</anti_patterns>

<error_recovery>
If the generator can't find the objective directory: emit `{ error: 'objective not found', objective: '<input>' }` and exit gracefully. Do NOT throw.

If a TRD has `type: ui` but missing `must_haves.artifacts`: skip that TRD with a note in the generator's stderr, do NOT abort. The planner (TRD 10-03) was supposed to enforce this — if it slipped through, the generator gracefully degrades.

If `.maestro/` directory doesn't exist: just emit zero Maestro rows. The bootstrap detector (TRD 10-04a) handles the missing-Maestro case at executor start.
</error_recovery>

## GREEN phase (commit 2: `feat(10-06): implement generateUAT pure function (GREEN)`)

Create `uat-generator.cjs`:

```javascript
'use strict';
const fs = require('fs');
const path = require('path');
const { output } = require('./helpers.cjs');

const STATE_DESCRIPTIONS = {
  loading: 'a loading spinner is visible and no other content is rendered',
  data: 'content renders correctly with no spinner or error overlay',
  error: 'an error message is shown with a retry option (or equivalent UX)',
  empty: 'an empty-state message is shown (no spinner, no error)',
  initial: 'the initial UI is visible (no user action taken yet)',
};

/**
 * Pure function: generate UAT.md content from a parsed objective's TRDs + Maestro flows.
 *
 * Per user correction (2026-05-24): web is DEFAULT coverage (not opt-in).
 * - State rows expand per (state, platform) — e.g., platform=[mobile, web] doubles rows.
 * - Maestro rows are MOBILE-ONLY by design (mobile-dev-inc/maestro#2591). No maestro_web branch.
 * - Web verification rows use `flutter drive` (per TRD 10-02's "Web verification mechanism").
 *
 * @param {object} input
 * @param {string} input.objective - objective slug
 * @param {string[]} input.sourceFiles - SUMMARY.md filenames
 * @param {object[]} input.trds - parsed TRD frontmatter objects (must_haves, type, platform, etc.)
 * @param {string[]} input.maestroFlows - paths to .maestro/*.yaml flow files
 * @returns {string} UAT.md content
 */
function generateUAT({ objective, sourceFiles, trds, maestroFlows }) {
  const now = new Date().toISOString();
  const flutterTrds = (trds || []).filter(t => t.type === 'ui' && t.stack === 'flutter');

  const rows = [];
  let n = 1;

  // State-coverage rows: one per (artifact, state, platform)
  for (const trd of flutterTrds) {
    const platforms = trd.platform || ['mobile', 'web'];  // default per TRD 10-03 planner gate
    const artifacts = (trd.must_haves && trd.must_haves.artifacts) || [];
    for (const art of artifacts) {
      for (const state of art.states || []) {
        for (const platform of platforms) {
          const desc = STATE_DESCRIPTIONS[state] || `the \`${state}\` state UI is rendered correctly`;
          const artBase = path.basename(art.path || 'artifact', '.dart');
          rows.push(`### ${n}. ${artBase} renders \`${state}\` state correctly on ${platform}
expected: Open the screen on ${platform}, trigger ${state} state, verify ${desc}.
result: [pending]
`);
          n++;
        }
      }
    }
  }

  // Maestro flow rows — MOBILE-ONLY by design. Iterate over all .maestro/ files referenced.
  const maestroFlowsList = maestroFlows || [];
  for (const flow of maestroFlowsList) {
    const flowBase = path.basename(flow);
    rows.push(`### ${n}. Maestro flow: ${flowBase} (mobile)
expected: Run \`maestro test ${flow}\` on a mobile device, verify all assertions pass.
result: [pending]
`);
    n++;
  }

  // Web-integration rows — one per artifact when web is in platform.
  for (const trd of flutterTrds) {
    const platforms = trd.platform || ['mobile', 'web'];
    if (!platforms.includes('web')) continue;
    const artifacts = (trd.must_haves && trd.must_haves.artifacts) || [];
    for (const art of artifacts) {
      const integrationPath = art.tests && art.tests.integration;
      if (!integrationPath) continue;  // no integration test declared
      const artBase = path.basename(art.path || 'artifact', '.dart');
      rows.push(`### ${n}. Web integration: ${artBase}
expected: Run \`flutter drive --driver=test_driver/integration_test.dart --target=${integrationPath} -d chrome\` and verify all assertions pass (chromedriver must be running on port 4444).
result: [pending]
`);
      n++;
    }
  }

  const total = rows.length;

  const frontmatter = `---
status: testing
objective: ${objective}

function cmdGenerateUAT(cwd, objectiveArg, raw) {
  if (!objectiveArg) {
    output({ error: 'objective argument required' }, raw);
    return;
  }
  const { extractFrontmatter } = require('./frontmatter.cjs');
  const objectivesRoot = path.join(cwd, '.planning', 'objectives');
  const padded = String(objectiveArg).padStart(2, '0');
  const dirs = fs.readdirSync(objectivesRoot).filter(d => d.startsWith(padded + '-') || d === objectiveArg);
  if (dirs.length === 0) {
    output({ error: 'objective not found', objective: objectiveArg }, raw);
    return;
  }
  const objDir = path.join(objectivesRoot, dirs[0]);

  // Gather TRDs and source SUMMARYs
  const trds = [];
  const sourceFiles = [];
  for (const f of fs.readdirSync(objDir)) {
    if (f.endsWith('-TRD.md')) {
      const content = fs.readFileSync(path.join(objDir, f), 'utf-8');
      const fm = extractFrontmatter(content);
      trds.push(fm);
    } else if (f.endsWith('-SUMMARY.md')) {
      sourceFiles.push(f);
    }
  }

  // Gather Maestro flows from cwd's .maestro/ dir
  const maestroDir = path.join(cwd, '.maestro');
  const maestroFlows = [];
  if (fs.existsSync(maestroDir)) {
    for (const f of fs.readdirSync(maestroDir)) {
      if (f.endsWith('.yaml')) {
        maestroFlows.push(path.join('.maestro', f));
      }
    }
  }

  const uat = generateUAT({ objective: dirs[0], sourceFiles, trds, maestroFlows });

  // Safety check: don't overwrite a user-edited UAT.md
  const uatPath = path.join(objDir, `${dirs[0]}-UAT.md`);
  if (fs.existsSync(uatPath)) {
    const existing = fs.readFileSync(uatPath, 'utf-8');
    if (/result:\s*(pass|issue|skipped)/.test(existing) || /status:\s*(complete|diagnosed)/.test(existing)) {
      output({ error: 'UAT.md already in use; refusing to overwrite', uat_path: uatPath }, raw);
      return;
    }
  }

  fs.writeFileSync(uatPath, uat, 'utf-8');
  output({ generated: true, uat_path: uatPath, test_count: (uat.match(/^### \d+\./gm) || []).length }, raw);
}

module.exports = { generateUAT, cmdGenerateUAT };
```
